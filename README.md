# Note Forge X Static

DB不要、環境変数不要、GitHub Pages向けの静的Webアプリ版です。

## この版の特徴

- GitHubにアップしてそのまま公開しやすい
- 8桁数字でワークスペースを識別
- 履歴は `localStorage` 保存
- APIキーは `sessionStorage` にのみ保存
- タブやブラウザを閉じるとAPIキーは消える
- スマホ対応
- リサーチ / 構成 / タイトル / 全文生成を1画面で実行

## 向いている使い方

- 購入者に「難しいセットアップなし」で使ってもらいたい
- DB接続や環境変数を避けたい
- まずはGitHub Pagesで配布したい
- 履歴はその端末だけで十分

## 注意点

- 履歴は**その端末・そのブラウザだけ**に残ります
- SafariやChromeでサイトデータを削除すると履歴も消えます
- 8桁コードは**本格的な認証ではなく識別キー**です
- Gemini Direct が最も相性が良いです
- OpenAI系はブラウザ直打ち公開運用に向かないため、互換エンドポイント欄は「自前プロキシ / 互換サービス」向けです

## GitHub Pages 公開手順

1. このフォルダ一式をリポジトリにアップ
2. GitHub の `Settings` → `Pages`
3. `Deploy from a branch` を選択
4. Branch は `main`、Folder は `/ (root)` を選択
5. 保存
6. 数十秒〜数分後に公開URLが発行される

## ローカルで確認するだけなら

`index.html` をブラウザで開くだけでも動きます。
ただし、環境によってはブラウザのセキュリティ制約で挙動差が出ることがあるため、公開URLでの確認も推奨です。

## データ保存仕様

### APIキー
- 保存先: `sessionStorage`
- 目的: このタブ中だけ生成可能にする
- ブラウザ終了で消去

### 履歴
- 保存先: `localStorage`
- 保存単位: 8桁コードごと
- 保存内容: 案件フォーム内容 / research / outline / hooks / article

## 推奨販売文脈

このアプリは「AIライター」ではなく、
**note収益化のためのリサーチ・設計・執筆OS** として見せる方が高単価化しやすいです。

訴求例:
- DB構築なしですぐ使える
- 購入者自身のGeminiキーで運用できる
- 端末保存なので導入が軽い
- スマホだけでも作業しやすい
- リサーチから長文生成まで一体化
